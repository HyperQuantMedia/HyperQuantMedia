# resources/ — brand assets

Unified-naming logo library. Three families:

## `logo-mark-*` — the bare mark (icon, no text)

| File | Background | Use |
|---|---|---|
| `logo-mark-transparent.png` | transparent, tight crop | **profile header**, anywhere over a colored/unknown bg |
| `logo-mark-1024-padded.png` | transparent, padded | large transparent placements with breathing room |
| `logo-mark-256.png` / `-512.png` / `-1024.png` | solid white | over dark/photo backgrounds where a white plate is wanted |

## `app-icon-*` — app/favicon tiles (mark on dark navy rounded square)

| File | Size | Use |
|---|---|---|
| `app-icon-16.png` | 16px | favicon |
| `app-icon-32.png` | 32px | favicon |
| `app-icon-64.png` | 64px | small app icon |
| `app-icon-128.png` | 128px | app icon, social avatar |

## `logo-wordmark-*` — mark + "HyperQuant Media" wordmark (white text)

| File | Dimensions | Use |
|---|---|---|
| `logo-wordmark-545x256.png` | 545×256 | dark-background headers (small) |
| `logo-wordmark-1089x512.png` | 1089×512 | dark-background headers (large) |

> Wordmark text is **white** — only shows on dark backgrounds.

## Naming rules

- lowercase, hyphen-separated: `<family>-<descriptor>[-<size>]`.
- families: `logo-mark`, `app-icon`, `logo-wordmark`.
- size suffix where multiple sizes exist (`-256`, `-512`); dims for wordmarks (`-545x256`).
